## 📋 最佳实践提醒

### ✅ **应该做的**

1. **经常提交**：完成一个小功能就提交
2. **写清楚提交信息**：使用 `feat:`, `fix:`, `docs:` 等前缀
3. **提交前检查**：运行 `git status` 和 `git diff`
4. **保护敏感信息**：永远不要提交 API keys、密码
5. **使用 .gitignore**：忽略不必要的文件

### ❌ **不应该做的**

1. ❌ 提交 `node_modules/`
2. ❌ 提交 `.env.local` 等敏感文件
3. ❌ 提交 `.DS_Store` 等系统文件
4. ❌ 提交构建产物 `dist/`, `build/`
5. ❌ 使用模糊的提交信息（如 "update", "fix"）
